The purpose of this project is to create 2 types of grid view of a particular kind of data. 1st one is detailed view and second one is compare view.

## Detailed view
Detailed view is a grid type of view with cells or input number types. There will be total 6 columns in the screen which full fit to the screen. At the left end, there will be Subjects column. In the header next to the name subject, there should be a sort icon. So the subjects will be listed on most left coloumn. Subjects are divided in to certain catgories. Fro example, physics and chemistry comes under science and english and french comes under languages. So this category name should be at the left end of first column 90 degree rotate to anti clockwise. category name's column should extend over the subjects it has. If it has more no:of subjects, it should expand over that. If there is minimum no:of subjects, for example one subject, then also this should not make any misalignment.
The second column to 6th column is for students. So the header of these coloumns will have student names(first row in header), the next row inside the student header should have the current, short term and mid term (vertically dividing that colum into 3). The next row in header should have option to display percentages for current, short term and mid term. the next row in student header should show texts basic, advanced and expert. As basic comes vertically under the current column. Both the header of Subject and header of student should have same height. Against each subjects displayed, there should be input fields of type number for basic advanced and expert for each students.
When user click on sort functioanlity on subjects header, it should sort within the category. The category shoukd no change position. It should  sort (asc, desc and back to no sort) within a category. When sorting happens, right hand side with input fields should also change. for example, if the subject physics goen down because of sorting, the right side values given in input fields should also go down and should be aligned with the subject physics.
Overall the width should be taken as full width of parent container. If there are lot of subjects, the body of this grid should scroll vertically.
Alignment of each subject to the columns under basic, advanced and expert should be correct in all screen sizes. This grid with 1 subject coloumn and 5 student column should be displayed as same for all screen sizes(laptops and desktops). Horizontal line after each subject which extends to the  students coloumn should be there. A separate separation after a category is over should also be there. If there are marks for the subjects, that also should be displayed in the input type number columns.

## Compare view
Compare view is a grid type of view. it has max of 6 coloumns. first coloumn header(label) is 'skill profiles'. There is a sort icon next to header name(similar to header name - subjects of detailed view). First colum will list the skill profile names. name can extend to 2nd line as well. If it overflows the 2nd line also, it should show ellipsis and show name on tooltip. 2nd column to 6th column represents the students. So the headers of that should have name, next to that, an edit icon, next to that a person icon. Each skill profile should have basic, advance and expert as first row. It should have the following text labels in 2nd row - '+1 level', 91, 80 ( each row is horizontally divided in students side, but it together constitute one row of skill profile). This is against each person for each skill profile. Sort button should have similar sort functionality as given for detailed view but for skill profile names here. The corresponding right hand side value should also change when sorting is done. Color theme can be as same as compare view.
